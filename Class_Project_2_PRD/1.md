# PRD 1 — Smart Job Tracker Dashboard (React Project)

## 1. Product Title

**Smart Job Tracker Dashboard**

---

# 2. Problem Statement

Job seekers today apply to dozens or even hundreds of jobs across platforms such as LinkedIn, company career pages, referrals, and job boards. Managing all these applications manually becomes extremely difficult.

Common problems faced by job seekers:

* Forgetting where they applied
* Missing interview schedules
* Losing track of follow-ups
* No visibility into job search progress
* Difficulty comparing offers or salary ranges

Most people use spreadsheets or notes, which lack automation, filtering, analytics, and proper organization.

The goal of this project is to build a **Smart Job Tracker Dashboard** using React that helps users:

* Track all job applications
* Monitor interview progress
* Manage upcoming interviews
* Search, filter, and sort applications
* View job search analytics

This application should simulate a **real SaaS productivity tool**.

---

# 3. Product Goals

### Primary Goals

The system should allow users to:

1. Add and manage job applications
2. Organize applications by stage
3. Quickly search applications
4. Filter and sort job entries
5. View analytics of their job search progress

---

### Secondary Goals

* Provide clean UI with dashboards
* Practice advanced React architecture
* Implement proper state management

# 4. Target Users

### Primary Users

* Students applying for placements

* Freelancers tracking job leads

### User Pain Points

Users need:

* centralized job tracking
* interview reminders
* easy search of applications
* analytics of job pipeline

# 5. Core Functional Requirements

---

# Feature 1 — Add Job Application

Users should be able to add a new job application.

### Fields required

* Company Name
* Job Role
* Location
* Salary Range
* Application Platform
* Status
* Applied Date
* Interview Date
* Notes

---

### Validation Rules

* Company name required
* Role required
* Applied date required

---

### Form Handling

Use:

```
react-hook-form
```

Validation using:

```
yup
```

---

# Feature 2 — View Applications List

Display all applications in a table or card view.

Each entry should show:

* Company logo
* Company name
* Role
* Status
* Applied date
* Salary range

---

### Actions available

* Edit
* Delete
* Bookmark

---

# Feature 3 — Search

Users should be able to search applications.

Search fields:

* company name
* role

Search should update results **dynamically while typing**.

---

### Performance Improvement

Use a **debounce search hook**.

Example custom hook:

```
useDebounce(searchTerm, 500)
```

---

# Feature 4 — Filters

Filters should include:

* Status
* Platform
* Location type

Example:

```
Applied
Interviewing
Rejected
Offer
```

---

# Feature 5 — Sorting

Users should be able to sort applications.

Sorting options:

* Applied Date
* Salary
* Company Name

---

# Feature 6 — Tabs for Job Pipeline

Applications should be categorized into tabs:

* Applied
* Interview Scheduled
* Offer Received
* Rejected

Each tab displays relevant applications.

---

# Feature 7 — Bookmark Important Jobs

Users should be able to bookmark jobs.

Bookmarked jobs appear in a special list.

---

# Feature 8 — Dashboard Analytics

Dashboard should display statistics.

Example metrics:

* Total applications
* Interviews scheduled
* Offers received
* Rejections

Charts should include:

* Pie chart of application stages
* Monthly applications graph
